# Contributing

Thanks for improving a Silesian Solutions project.

1. Read the repository README and local contribution instructions first.
2. Keep changes focused. Discuss large, breaking, or architectural changes in an issue before implementation.
3. Follow the repository's existing style and tooling. Prefer Conventional Commit-style PR titles, for example **feat: add export** or **fix: handle empty input**.
4. Run the relevant tests, linters, type checks, and builds locally.
5. Update documentation and the changelog when behavior changes.
6. In the pull request, explain the problem, the solution, and exactly how the change was verified.

Never commit credentials, tokens, production data, or customer information. Report vulnerabilities privately according to [the security policy](SECURITY.md).
